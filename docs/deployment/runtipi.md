# Deployment: Runtipi

[Runtipi](https://runtipi.io) is a self-hosted app manager for home servers and Raspberry Pis. HopLedger Backend ships a built-in app definition so you can install it in a few clicks.

## Prerequisites

- Runtipi v3.0.0 or newer installed and running on your server
- This repository accessible (public GitHub repo)

## Docker Image

Every push to `main` automatically builds and publishes a multi-arch Docker image (`linux/amd64` + `linux/arm64`) to GitHub Container Registry:

```
ghcr.io/haertibraeu/hopledger-backend:latest
```

> **After the first CI run**, make the package public:  
> GitHub → organization settings → Member privileges → enable **"Allow members to change package visibility"** → Save.  
> Then go to the package settings and set visibility to **Public**.

## Installation

### 1. Add this repository as a custom app store

In Runtipi: **Settings → App Stores → Add Store**

```
https://github.com/HAERTIBRAEU/HopLedger-Backend
```

### 2. Install the app

Go to **App Store → Custom → HopLedger Backend** and click **Install**.

Fill in the form fields:

| Field | Description |
|-------|-------------|
| **API Key** | A strong secret — generate one with `openssl rand -hex 32`. Leave empty to disable auth. |
| **Show Locations** | Whether the public `/api/public/inventory` endpoint includes location names. |

The database password is auto-generated by Runtipi — no manual setup needed.

### 3. Migrations

Database migrations run automatically on every container start — no manual steps required.

## API Key Rotation

1. In Runtipi, open the app → **Settings**.
2. Update the **API Key** field.
3. Restart the app.
4. Update the same key in your Android app under **Settings → API Key**.

## Seed Default Categories

Default expense categories are not seeded automatically. Run once after the first deploy if needed:

```bash
docker exec -it hopledger-backend npx tsx prisma/seed.ts
```

## Updating

Runtipi will notify you when a new version is available. Click **Update** in the app panel — Runtipi pulls the latest image and restarts the container automatically.
